## ICI Tier Classification System

When working with the data, understand the scoring methodology:

| Tier | Points | Description |
|------|--------|-------------|
| Tier 4 | ±4 | Laws affecting many aspects of immigrant life (highest impact) |
| Tier 3 | ±3 | Laws affecting crucial aspects (difficult to avoid/substitute) |
| Tier 2 | ±2 | Important but not crucial aspects (alternatives exist) |
| Tier 1 | ±1 | Less significant impacts (e.g., English-only laws) |

*Positive points indicate immigrant-friendly legislation; negative points indicate restrictive legislation.*

## Guidelines for AI Assistants

### DO:
- **Treat data files carefully** - The Excel files contain research data; preserve their integrity
- **Understand this is academic work** - Changes should maintain scholarly tone and accuracy
- **Reference the methodology** - Any analysis should align with the ICI tier classification system
- **Keep the HTML self-contained** - If editing index.html, ensure all resources remain embedded
- **Preserve base64-encoded images** - These are embedded within the HTML

### DON'T:
- **Don't regenerate the HTML** without the original source files
- **Don't modify raw data** in Excel files without explicit instruction
- **Don't break embedded resources** - The HTML relies on base64-encoded assets
- **Don't add external dependencies** - The project is designed to be fully self-contained

### Working with Data Files

The Excel databases contain:
- **State data (2005-2020):** `ACTIVE State_Legislation_Database.xlsx`
- **Local data:** `ACTIVE_Local_Legislation_Database.xlsx`

When analyzing or referencing this data:
1. Understand the tier classification before interpreting scores
2. Note temporal patterns, especially around 2017 ("Trump Effect")
3. Consider geographic variations (e.g., California vs. Arizona patterns)

### HTML Modifications

The `index.html` file is very large (~1.7 MB) with long lines due to embedded resources. When editing:

1. Use targeted searches (grep) rather than reading the entire file
2. Make precise, minimal changes
3. Avoid reformatting that could break embedded data
4. Test changes locally before committing

## ICI Data Explorer (Chatbot)

The `chatbot.html` file provides an interactive interface for exploring the legislation database.

### Features
- **Natural language queries** - Ask questions like "Show laws in California" or "Compare Texas and Florida"
- **Quick filters** - Filter by state, type, year range, and direction (positive/restrictive)
- **Visualizations** - Interactive charts using Chart.js
- **Data export** - Export filtered results to CSV
- **Trend analysis** - View legislation trends over time

### Supported Query Types
- State-specific: "Show me all laws in CA" or "California legislation"
- Year-specific: "Laws passed in 2017"
- Type-specific: "Show policing laws" or "sanctuary policies"
- Comparisons: "Compare California and Texas"
- Trends: "Show trends over time"
- Trump Effect analysis: "Show 2017 Trump effect"

### Data Files
- `ici_data.json` - JSON export of both Excel databases, loaded by the chatbot
- Contains 1,910 state laws and 2,618 local laws
